# Contributing to Whole

This project is built for patients first. If you're contributing, keep that in your mind the entire time.

## What's in scope

- Making Tier 1 (the layperson version) easier and more reliable
- Improving provider coverage and connection guides
- Making the AI layer work with more local models
- Translations — English is not everyone's first language, and this matters globally
- Accessibility fixes — anything that makes this harder for people with limited energy or mobility is a bug

## What's out of scope

- Features that require PHI to leave the user's machine
- Anything that introduces a dependency on a paid cloud service
- Complexity that raises the barrier for non-technical patients

## How to submit changes

Fork, branch, pull request. Keep PRs small and scoped. Describe what problem the change solves for a patient — not just what the code does.

## A note on tone

The README is written for people who are sick and tired. Not "tech-tired" — literally ill. Keep that register when you write docs, error messages, or instructions. No jargon that isn't explained. No walls of text. Short paragraphs. White space is part of the writing.
